<H2 LANG="en-GB" CLASS="western" STYLE="page-break-before: always">II.
Systems installation</H2>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>It is recommended to
change the default partition suggestions.</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>For example:</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>/dev/hda1	swap		2x RAM</FONT></P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>/dev/hda2	/		1.0GB</FONT></P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>/dev/hda3	Extended</FONT></P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>/dev/hda5	/var		251 MB</FONT></P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>/dev/hda6	/tmp		251 MB</FONT></P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>/dev/hda7	/usr		2.0GB</FONT></P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>/dev/hda8	/data		rest</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>This should be enough
for a later X installation, if desired. But I recommend to choose
<I>Minimal Installation</I>, which is without any graphical
interface.</FONT></P>

</P>

<H4 LANG="en-GB" CLASS="western">PostgreSQL</H4>
<P LANG="fr-FR" CLASS="western"><FONT SIZE=3>PostgreSQL 7.2 is
supported by UMN-MapServer, but version 7.3 is really recommended and
since PostGIS 0.8 also PostgreSQL 7.4 is supported.</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>The source code is
needed for building PostGIS against the PostgreSQL source tree later
on. It  is installed into <SPAN STYLE="background: transparent">/usr/src/packages/SOURCES/</SPAN>
and needs to be extracted and compiled.</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>	# bunzip2
postgresql-7.3.4.tar.bz2</FONT></P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>	# tar -xf 
postgresql-7.3.4.tar</FONT></P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>	# cd postgresql-7.3.4/</FONT></P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>	# ./configure
--without-readline (if not installed on the system)</FONT></P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>	# make</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3><SPAN STYLE="background: transparent">To
start postgresql manually type <I>/etc/init.d/postgresql start</I> on
the command line. To start postgresql (or any other daemon) on boot,
call the <I>Runlevel Editor</I> within Yast2.</SPAN></FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>To have PostgreSQL
accepting Internet connections <I><FONT COLOR="#ff0000">/var/lib/pgsql/data/postgresql.conf</FONT>
</I><SPAN STYLE="font-style: normal">needs</SPAN> to be edited.
'<FONT COLOR="#ff0000">tcpip_socket=false</FONT>' or 'tcpip_socket=0
needs to be uncommented and set to <FONT COLOR="#ff0000">true</FONT>
( or '1'  for pgsql 7.2.x).</FONT></P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>It is also required to
configure the pg_hba.conf, which contains the list of allowed hosts. </FONT>

<H3 LANG="en-GB" CLASS="western">PostGIS 0.7.5/0.8.1/0.9.0</H3>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>PostGIS needs to be
uncompressed and untared in the archive in the &quot;contrib&quot;
directory of the PostgreSQL source tree. </FONT>
</P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.04cm"><FONT SIZE=3>#
cd /sourc-tree-of/postgresql-7.<SPAN STYLE="font-style: normal">3.4</SPAN>/contrib
</FONT>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.04cm"><FONT SIZE=3>#
gzip -d -c postgis-0.7.5.tar.gz | tar xvf -</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>Either the just created
directory has to be renamed</FONT></P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.06cm"><FONT SIZE=3>#
mv postgis-0.7.5 postgis </FONT>
</P>
<P LANG="en-GB" CLASS="western"><FONT SIZE=3>or t<SPAN STYLE="font-style: normal">he
Makefile has to be edited. For projection support leave the value for
</SPAN><I>USE_PROJ</I><SPAN STYLE="font-style: normal">  and
additionally since PostGIS 0.8 for geos support leave the value for
</SPAN><SPAN STYLE="background: transparent"><FONT FACE="Times New Roman, serif"><I>USE_GEOS
</I><SPAN STYLE="font-style: normal">(Requires a geos-installation)</SPAN><I>,
</I><SPAN STYLE="font-style: normal">otherwise change it to &ldquo;0&rdquo;.</SPAN></FONT></SPAN></FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.04cm"><FONT SIZE=3><I><B>#
</B></I><SPAN STYLE="font-weight: medium"><SPAN STYLE="font-style: normal">cd
postgis</SPAN></SPAN></FONT></P>
<P LANG="de-DE" CLASS="western" STYLE="margin-left: 2.04cm"><FONT SIZE=3>#
make</FONT></P>
<P LANG="de-DE" CLASS="western" STYLE="margin-left: 2.04cm"><FONT SIZE=3>#
make install</FONT></P>

<H4 LANG="en-GB" CLASS="western">Load PostGIS functions and objects
into PostgreSQL</H4>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>The file <I>postgis.sql
</I><SPAN STYLE="font-style: normal">uses the variable $libdir. This
variable contains the path to the PostGIS libraries. On some systems
it contains a wrong value and executing </SPAN><I>postgis.sql</I>
fails.</FONT></P>
<P LANG="de-DE" CLASS="western"><FONT SIZE=3>To correct this either
the string '$libdir' can be changed into the absolute path, which can
be found out via <SPAN LANG="en-GB"><FONT FACE="Times New Roman, serif"><I>pg_config
--pkglibdir</I> or the libraries can be copied to to the appropriate
location.</FONT></SPAN></FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT FACE="Times New Roman, serif"><FONT SIZE=3>The
libraries can be found in <I>/usr/local/pgsql/lib/</I><SPAN STYLE="font-style: normal">.</SPAN></FONT></FONT></P>
<P LANG="en-GB" CLASS="western"><FONT FACE="Times New Roman, serif"><FONT SIZE=3><I>pg_config
--pkglibdir</I> shows <I>/usr/lib/postgresql</I>.</FONT></FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT FACE="Times New Roman, serif"><FONT SIZE=3>ln
-s /usr/local/pgsql/lib/libpostgis.so.0.7 /usr/lib/postgresql</FONT></FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
cp /usr/local/pgsql/lib/libpostgis.a /usr/lib/postgresql (PostGIS
0.7)</FONT></P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
cp /usr/local/pgsql/lib/libpostgis.so.0.7 /usr/lib/postgresql
(PostGIS 0.7)</FONT></P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><BR>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
cp /usr/local/pgsql/lib/libpostgis.so.0.8 /usr/lib/postgresql
(PostGIS 0.8/0.9)</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><FONT FACE="Times New Roman, serif"><FONT SIZE=3>In
<I>pg_config --pkglibdir</I> (<I>/usr/lib/postgresql</I>) two
symbolic links have to be created.</FONT></FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western"><BR>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
ln &ndash;s libpostgis.so.0.7 libpostgis.so (PostGIS 0.7)</FONT></P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
ln &ndash;s libpostgis.so.0.7 libpostgis.so.0 (PostGIS 0.7)</FONT></P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><BR>
</P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
ln -s libpostgis.so.0.8 libpostgis.so (PostGIS 0.8/0.9 )</FONT></P>
<P LANG="en-GB" CLASS="western" STYLE="margin-left: 2.05cm"><FONT SIZE=3>#
ln -s libpostgis.so.0.8 libpostgis.so.0 (PostGIS 0.8/0.9)</FONT></P>
<P LANG="en-GB" CLASS="western"><BR>
